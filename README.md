# `HALTₜₘ` in `L∃∀N 4`

Formalizing the halting problem using the `SingleTapeTM` of Cslib.

Most of the work here modifies the structure in https://github.com/aalok-thakkar/undecidability/blob/Main/Halt/.

## Commit Guidelines
Every commit must keep `lake build` clear, except `sorry` warnings for definitions, NOT for theorems and lemmas.

Commit any changes made at logical intervals (e.g., don't commit incomplete features); keep the commit messages clear and concise. Follow the following format (*everything below this point was generated by Claude Sonnet 4.6*):

```
<type>(<scope>): <short description>
```

- `type`: what kind of change (see below)
- `scope`: optional, the affected module or file (e.g. `parser`, `lakefile`, `tactics`)
- `description`: imperative, lowercase, no trailing period

### Types

| Type | When to use |
|------|-------------|
| `init()` | Initial commit |
| `feat()` | New feature or tactic added |
| `fix()` | Bug or proof error fixed |
| `refactor()` | Code restructured without changing behavior |
| `style()` | Formatting, whitespace, or naming (no logic change) |
| `docs()` | Docstrings, comments, or README updated |
| `test()` | `#check`, `#eval`, `example`, or spec added/updated |
| `chore()` | Toolchain bump, dependency update, config change |
| `perf()` | Proof or computation made more efficient |
| `ci()` | GitHub Actions or CI pipeline changes |
| `build()` | `lakefile.toml`, or build system changes |
| `revert()` | Reverts a previous commit |

### Scopes

| Scope | Refers to |
|-------|-----------|
| `lakefile` | `lakefile.lean` or `lakefile.toml` changes |
| `toolchain` | `lean-toolchain` version bump |
| `mathlib` | Mathlib dependency or import changes |
| `tactics` | Custom tactic definitions |
| `parser` | Syntax or parser extensions |
| `notation` | Custom notation definitions |
| `core` | Root module or shared logic |

### Examples

```
init()
feat(tactics): add custom simp lemma for nat arithmetic
fix(parser): handle empty token stream edge case
refactor(core): extract evaluation logic into Eval.lean
style(notation): align operator precedence declarations
docs(readme): add build and usage instructions
test(tactics): add #check examples for ring_nf usage
chore(toolchain): bump lean-toolchain to v4.28.0
chore(mathlib): update mathlib4 to latest stable
build(lakefile): add new Lean.Parser dependency
perf(core): replace List with Array in hot path
ci: add lake build step to GitHub Actions workflow
revert: feat(parser): add support for unicode identifiers
```

One logical change per commit; if you find yourself writing "and" in the message, consider splitting it.
- Scope to the Lean module name when possible: `feat(MyProject.Eval): ...`
- Never commit directly to `main`. Commit to a new branch.
